## For loops can be optimized to save gas
In line , https://github.com/code-423n4/2022-08-nounsdao/blob/c1c7c6201d0247f92472419ff657b570f9104565/contracts/governance/NounsDAOLogicV2.sol#L292, the for loop can be optimized as :
```
uint256 len = proposal.targets.length;
for (uint256 i; i < len;) {
            queueOrRevertInternal(
                proposal.targets[i],
                proposal.values[i],
                proposal.signatures[i],
                proposal.calldatas[i],
                eta
            );
           unchecked {
           ++i;
           }
        }
```

## Caching of storage variable to save gas:
In line : https://github.com/code-423n4/2022-08-nounsdao/blob/c1c7c6201d0247f92472419ff657b570f9104565/contracts/governance/NounsDAOLogicV2.sol#L350-L354, `proposal.proposer` is used twice, hence it can be first saved in a memory variable and the use it, which would save gas for 1 storage access.
Similarly in line https://github.com/code-423n4/2022-08-nounsdao/blob/c1c7c6201d0247f92472419ff657b570f9104565/contracts/governance/NounsDAOLogicV2.sol#L443, `proposal.forVotes` can be cached.
